Knowledge Engine Core - Pain Points and Later Fixes
===================================================

Purpose
-------
This file tracks annoyances, rough edges, bugs, and developer-experience
problems that are not blocking Phase 0, but should be fixed in later updates.

Labels use the same names that should be created in GitHub after the repository
is published.

High Priority
-------------

1. Poetry install path needs more hardening on Windows.
   Label: `developer-experience`, `windows`, `setup`
   Impact: Contributors are expected to use Poetry, but Poetry dependency
   resolution currently fails on this machine with certificate verification
   errors against `pypi.org`.
   Later fix: Add a Windows setup guide and/or bootstrap script that installs
   Poetry, verifies certificate behavior, and runs the project checks.

2. Poetry 2.4.1 installed and launches, but dependency resolution fails with
   `SSLCertVerificationError: unable to get local issuer certificate` against
   `https://pypi.org/simple/ruff/`.
   Label: `developer-experience`, `dependencies`, `security`
   Impact: The documented Poetry install path may fail on this machine until
   Poetry networking/certificate/proxy behavior is corrected. Pip-based virtual
   environment installation works under Python 3.14.6.
   Later fix: Diagnose Poetry's PyPI access, certificate store, and proxy
   configuration. Keep a pip-based verification fallback documented until the
   Poetry path is fully reliable.

3. No import manifest exists yet.
   Label: `phase-1`, `ingestion`, `data-quality`
   Impact: Bulk ingestion will be hard to audit and resume.
   Later fix: Add import runs, source manifests, duplicate reports, and failure
   logs.

Medium Priority
---------------

4. Machine-level PATH still contains stale Python312 entries.
   Label: `developer-experience`, `windows`, `setup`
   Impact: Python 3.12.10 was broken and has been uninstalled, but the stale
   machine PATH entries require administrator rights to remove. They do not
   currently shadow Python 3.14 because the Python312 executable is gone.
   Later fix: Remove these stale machine PATH entries from an elevated shell or
   Windows Environment Variables UI.

5. Typer 0.12.x is not compatible with the newest Click versions observed
   during validation.
   Label: `dependencies`, `cli`, `maintenance`
   Impact: CLI help failed with Click 8.4.2.
   Current mitigation: Project pins Click to `>=8.0,<8.2`.
   Later fix: Re-test and loosen the pin after upgrading Typer.

6. The first generated PDF parser is best-effort only.
   Label: `parser`, `data-quality`, `phase-1`
   Impact: Title, author, abstract, and DOI extraction can be weak for real
   scientific PDFs with complex layouts.
   Later fix: Add metadata enrichment from PubMed/Crossref and stronger parsing
   tests using a small legal fixture corpus.

Low Priority
------------

7. Search snippets currently depend on SQLite FTS behavior.
   Label: `search`, `user-experience`
   Impact: Snippet quality may vary by query and document structure.
   Later fix: Add deterministic snippet generation around matched terms.

8. No sample papers are committed.
   Label: `documentation`, `testing`, `good-first-issue`
   Impact: New contributors cannot try realistic import/search behavior
   immediately after install.
   Later fix: Add a legally redistributable sample PDF or a fixture generator
   command.

Future Enhancement
------------------

9. Real-world corpus validation is not yet available.
   Label: `phase-1`, `corpus`, `research`
   Impact: Phase 0 tests prove the mechanics, but not behavior across a real
   scientific corpus.
   Later fix: Build a legally usable focused corpus and track parser/search
   failures as structured issues.

Completed During Hygiene Pass
-----------------------------

10. No CI pipeline exists yet.
    Label: `ci`, `developer-experience`
    Resolution: Added `.github/workflows/quality.yml` to run black, ruff, mypy,
    and pytest on pushes to `main` and pull requests.
