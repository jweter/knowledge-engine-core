# v0.2.0-alpha.6 — Metadata Enrichment Preview

## Added

- provider-neutral metadata query, candidate, diagnostic, and result contracts;
- deterministic candidate normalization, validation, and conflict classification;
- Crossref work-response parsing for DOI, title, journal, publication year, authors, and ISSN;
- an injected Crossref provider transport boundary;
- a standard-library HTTPS-only transport with host, redirect, timeout, and response-size controls;
- `ke metadata-preview --doi <doi> --provider crossref`;
- explicit candidate, no-match, and provider-failure CLI output;
- an authoritative error-resolution ledger for recurring CI failures.

## Safety and compatibility

- normal corpus ingestion remains offline by default;
- metadata preview performs no database writes;
- curated, parser-derived, and preferred metadata are not overwritten;
- provider failures do not alter import or paper state;
- required tests make no live provider calls;
- no schema migration is included in this release.

## Persistence decision

External metadata candidates remain transient in this release. A normalized persistence model will be reconsidered only after durable audit, review, or promotion access patterns are defined.
